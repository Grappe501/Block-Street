## ACN-M01 — Purpose

**[ACN-M01]** The **Action Constitution** establishes the philosophy, principles, and operational standards for how work is **planned, coordinated, executed, evaluated, and preserved** throughout the Arkansas Student & Youth Organizing Network.

**[ACN-M01a]** Every mission, project, team, committee, campus, county, and statewide initiative operates within this shared framework.

**[ACN-M01b]** The platform exists to help people **accomplish meaningful work together** — not to manage tasks for their own sake.

**[ACN-M01c]** AOS transforms **ideas into completed community work**. Every mission, project, task, and event **inherits** this execution philosophy [CCN-M02 parallel].

**[ACN-M01d]** AOS is the engine that makes Phase 4 community infrastructure **operational** — without it, communities have systems but no execution spine.

**[ACN-M05b]** **Purpose always comes before process** [ACN-M18].

## ACN-M04 — Civic Operating Loop Architecture

**[ACN-M04]** **Civic Operating Loop** — signature architectural concept of AOS.

**[ACN-M04a]** Everything built in Phases 1–4 forms a **continuous cycle**:

```text
People → Relationships → Communities → Teams
        → Missions → Projects → Tasks → Events
        → Impact → Knowledge → Legacy → New People
```

**[ACN-M04b]** The system **loops**. Every completed mission creates:

- Better playbooks [CKLS-001]
- Better mentors [PGL-001, SCN-001]
- Better communities [CGS-001]
- Better leaders [CLS-001]
- Better stories [CST-001 → CLS-001]
- Better opportunities [OEX-001]

**[ACN-M04c]** The platform **learns** — every year becomes smarter because every community contributes back to the shared knowledge and capability network [CCE-001, CKLS-001].

**[ACN-M04d]** Orchestrator concept: `closeCivicOperatingLoop(missionId)` — impact → knowledge → legacy → new opportunities.

## ACN-M26 — Mission Operating Record (MOR)

**[ACN-M26]** **Mission Operating Record (MOR)** — signature operational concept of the Action Constitution.

**[ACN-M26a]** Every mission **automatically generates** a Mission Operating Record — the complete operational history of that mission.

**[ACN-M26b]** The MOR includes:

| Section | Source module |
|---------|---------------|
| Mission purpose | MPS-001 · MDS-001 |
| Mission Canvas | MPS-M15 · MDS-001 |
| Team members | TWG-001 |
| Timeline | TSOS-001 |
| Tasks and milestones | EOS-001 |
| Volunteer participation | VDS-001 |
| Capacity coordinated | CCS-001 |
| Decisions made | CDS-001 |
| Communications | CCNET-001 |
| Photos and media | CST-001 |
| Community impact | CIIS-001 |
| Lessons learned | LIS-001 |
| Recognition | CRA-001 |
| Learning & Improvement | LIS-001 |
| Legacy links | CLS-001 |

**[ACN-M26c]** When the mission ends, the MOR is **archived — but never forgotten**.

**[ACN-M26d]** Future organizers search past MORs to answer:

- *How did another campus run this event?*
- *Who organized the first county cleanup?*
- *What challenges did they encounter?*
- *What would they do differently next time?*

**[ACN-M26e]** The MOR is the **operational memory** of the platform — combined with Community Brain [CKLS-001], Community Legacy [CLS-001], Capability Graph [CCE-001], and Personal Digital Twins [PDT-001], every mission leaves behind **knowledge that makes the next mission stronger**.

**[ACN-M26f]** Orchestrator concept: `generateMissionOperatingRecord(missionId)` — aggregates all operational artifacts; `archiveMissionOperatingRecord(missionId)` — closes MOR and feeds Civic Operating Loop.

**[ACN-M26g]** Database: `DB-MOR` · table: `mission_operating_records`

## ACN-M19 — How Work Should Happen

**[ACN-M19]** Inherited principles for all execution:

| Principle | Meaning |
|-----------|---------|
| Mission-first | No orphan tasks [MPS-001] |
| Finish what you start | Commitment & follow-through without punishment [CFS-001] |
| Preserve decisions | Decision history [CDS-001, CKLS-001] |

## ACN-BG — Burt Implementation Guidance

**[ACN-BG]** Implementation should:

1. **Use Action Constitution as governing framework** for all operational modules — like CCN-001 for communities
2. **Maintain consistent workflows** across every community [COS-M02 equal infrastructure]
3. **Separate philosophy from implementation details** — this doc is constitutional; modules are operational
4. **Support reusable mission templates** [MDS-001]
5. **Preserve operational history** via Mission Operating Record [ACN-M26]
6. **Ensure every completed action feeds** knowledge and legacy systems [CKLS-001, CLS-001, CCE-001]
7. **Enforce operational spine and Action hierarchy** — tasks link to milestones link to projects link to missions
8. **Implement Civic Operating Loop closures** — impact feeds knowledge feeds legacy feeds new opportunities
9. **Never invert ACN-M18** — process, status, paperwork serve people and purpose
10. **Build on Phase 4** — extend MPS, TSOS, CCE, CLS — do not fork parallel systems
11. **Auto-generate MOR** as missions progress — aggregate, don't duplicate module data
