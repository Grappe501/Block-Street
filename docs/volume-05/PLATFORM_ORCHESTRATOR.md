### Kernel Responsibilities

**Constitutional Enforcement** — Before significant actions occur, evaluates identity, permissions, governance, Business Rules, privacy, AI policies, and institutional policies.

**Context Engine** — Continuously maintains shared context: Participant · Community · Institution · Mission · Calendar · Workspace · Runtime · Device · LocalBrain · Accessibility preferences. No subsystem reconstructs context independently.

**Coordination Engine** — Coordinates Platform Services, AI Federation, Event Fabric, Synchronization Mesh, Communications, Knowledge Retrieval, and Runtime Federation without tight coupling.

**Resource Engine** — Works with Adaptive Capacity Grid to determine where work executes, which AI model responds, which runtime processes requests, which caches are used, and which connectors are available.

**Trust Engine** — Every action generates Trust Ledger entries, Community Event Ledger events, observability signals, security evaluations, and historical context.

**Learning Engine** — Continuously feeds Knowledge Observatory, Community Brain, Operational Intelligence, Executive Operations Center, and Platform Stewardship Office.
