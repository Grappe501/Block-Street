# Knowledge Evolution Protocol

**Protocol ID:** CAE-11.12-W7 · **Contract:** KNW-OPT-001

## Constitutional Boundaries

Wave 7 may identify improvements, create proposals, run pilots, measure outcomes, and recommend canonical revisions.

Wave 7 may never directly rewrite published knowledge, alter standards, publish AI content, erase history, or deploy AI changes without approval.

## Improvement Lifecycle

`detected → candidate → triage → proposed → review → pilot → adoption → implementation → outcome → institutionalized`

Rejected, deferred, and failed pilots remain in institutional history.

## Canonical Boundary

Approved improvements execute only through Wave 3 `knowledgeApplicationService.executeCommand`. W7 services set `direct_store_mutation: false`.

## Registry

Machine-readable improvement types: `data/phase-11/knowledge_improvement_registry.json`

## Prohibited Actions

16 rules in `OPTIMIZATION_PROHIBITED_ACTIONS` including `direct_canonical_mutation`, `self_implement_recommendation`, `train_model_automatically`, `create_human_score`.
