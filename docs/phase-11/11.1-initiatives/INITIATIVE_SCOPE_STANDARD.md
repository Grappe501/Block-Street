# Initiative Scope Standard

**Build:** 11.1 · **Wave:** W1 · **Canonical source:** [01_CONSTITUTION.md](01_CONSTITUTION.md)

## Scope Dimensions

| Dimension | Questions |
|-----------|-----------|
| Institutional | Governing institution, participating units, partner institutions |
| Human | Authorized groups, exclusions |
| Geographic | Neighborhood, city, county, region, state |
| Time | Start, review points, expected completion |
| Functional | Included work, excluded work |
| Data | Data used, ownership, access |
| Resource | Funds, facilities, staff, volunteers, vehicles, systems |
| Public | Private, internal, partner-visible, member-visible, public |

## In-Scope / Out-of-Scope

Every approved initiative must include explicit statements. Example:

**In scope:** Build pilot curriculum, recruit instructors, launch one cohort, evaluate outcomes.  
**Out of scope:** Statewide enrollment, accreditation, permanent faculty hiring.

## Material Scope Expansion

Requires: change request, rationale, impact review, approval, charter version, audit event.

Material expansion includes: new institution, geography, population, major budget, high-risk activity, public comms expansion, youth, sensitive data, legal obligation, extended timeline, new executive objective.

Enforcement: 11.1-W3 service blocks silent expansion (acceptance scenario 5).

## Purpose Model (Six Questions)

1. **Why** — Why does this initiative exist?
2. **Need** — What condition created it?
3. **Intended change** — What should be different on success?
4. **Beneficiaries** — Who benefits or participates?
5. **Institutional alignment** — Which mission or priority?
6. **Consequence of inaction** — What if the institution does nothing?

Problem statements describe current condition and affected population without prejudging solution.
