# Initiative Experience Constitution

**Requirement:** CAE-11.1-W4-UX-*

## UX Invariants

1. Every screen shows **institution** and **lifecycle state**
2. Consequential actions name **who may perform them** and **what will happen**
3. **Server reauthorizes** every command — hidden UI is not authorization
4. **Blocked actions** explain why and the next step
5. **No direct status editing** — only explicit lifecycle actions
6. **Visibility ≠ authority**
7. **Proposed owners ≠ active owners**
8. **AI suggestions** remain advisory (future charter assist)
9. Every primary screen shows **one clear next action**
10. History is **human-readable**, not raw audit dumps

## Progressive Disclosure

- **Immediate** — what the Human needs now
- **Contextual** — supporting detail for the current action
- **Advanced** — governance, configuration, technical audit (future)

## Language

Use **Our** / **Nuestro** in shared institutional copy. Core workflows support conversational Spanish via `ux/locale.ts`.
